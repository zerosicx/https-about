What is a Request?

A request needs to be sent from our computer to the web server to retrieve all the resources required to access/view/modify a web page.

Requests

The modern day request looks like this:

This is a request to get an html file from the example website we've been using so far.

Looks complicated; but it can be broken down into simple parts.

Header: this consists of logistical information such as the URL to retrieve the information from, the request command GET, and other information such as where to send the response back to.
Intentional Blank Line: This is used to separate the header from the body.
Body: can include additional information. Nothing special is used to simply view a website, but if we were to make a comment on a YouTube video, it would have that information.

Responses

Naturally if we make a request, we will get back a response. As per tradition, this is what a modern day response looks like:

This looks even more intimidating than before. It can get even longer, but for the scope of this information page, we want to understand the basics:

Header: logistical information varying from a status line, the type/length/age of data being sent back, all the way to where this data is being stored. Amongst other logistical things.
Intentional Blank Line: used to separate the header from the body.
Body: contains content of the requested item. For an HTML file this is the HTML code. Similar for CSS and JavaScript files. For images this might just consist of random bytes – sequences of 1’s and 0’s that our browser understands so we don’t have to.

Common Error Codes

Now that we know what requests are, we should know some of the most common errors that come with making a request. This knowledge can help us troubleshoot later.

Error Code Meaning
400 Bad request. A request not written correctly.
401 Unauthorised. The user trying to access the resource has not been authenticated/not authenticated correctly. Credentials required.
403 Forbidden. The request is valid, but the server is rejecting the request due to lack of access permissions.
404 The infamous NOT FOUND error. The user was able to communicate with the server, but the resource was not found.
500 Internal server error. Server cannot process the request due to unknown reasons. The most common being server misconfiguration or missing packages.
502 Bad gateway. The server is a gateway/proxy server and is not receiving a valid response from its backend servers that should fulfill the request.
504 Gateway timeout. The server is a server/proxy server and it’s unable to retrieve a response from the backend servers within an allowed time period.
