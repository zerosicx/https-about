Requesting a Website: Simplified

  1. You type a URL on the top search bar of your browser.
  2. Your browser then generates a client request to the web server you’re trying to reach. Your browser already knows where to send this request just from your URL!
  3. The web server receives it and sends it to the correct port to handle your request. For HTTP, this port is 80 and HTTPs, this port is 443. These are the commonly used ports that are just good to know. Remember that a port itself just stores the application that is going to handle your request.
  4. The server then sends back a response which looks like a request, but it stores information about the status of the request and is sent with the item requested.

Summary

In short, when we send a request, we get sent back all the page resources to build the page. Our browser automatically knows to either open a new tab or window, and the requested page is built piece by piece. The page is then displayed (rendered) to the user, you.
