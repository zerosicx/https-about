HTTP(s)

Hyper Text Transfer Protocol

HTTP is a transfer protocol – set of rules for a procedure – concerned with the transportation of a web page across the internet. Web pages are files themselves and are contained in HTML documents for content, CSS for styling, and JavaScript documents for functionality.

HTML: Hyper Text Markup Language

HTML Documents consist of bodies of text, images, videos, gifs, and other forms of media. When it was first created however, HTML only consisted of text. So, we’ve come a long way from 1993. Similarly, the first HTTP prototype released was in 1991! Currently we’re on HTML 5.

Protocol

A communication mechanism used to retrieve the resource. You could think of it as a methodology or a way of doing something. In this case, it is the way of transferring website building-blocks comprised of documents.

URL (Uniform Resource Locator)

a URL is what we commonly know as the website name or domain. It’s been greatly simplified for everyday use, but we can break it down to understand how we see websites today. For example:

  1. HTTPS:the protocol for retrieving the website
  2. The website domain: the name of the domain such as kmart.co.nz and github.io
  3. Port: right after the domain name, a hidden protocol is there but does not need to be explicitly stated.
  4. File path: the path in the file system of a webserver that leads to the file
  5. File name

You may have noticed the protocol here is HTTPs rather than the standard HTTP. The “s” in HTTPs simply means secure. The whole of the internet has recently moved to this protocol.

Secure means all our website traffic is encrypted. This makes it harder to track our data and identify us, and our location on the internet. For this reason, always make sure you’re on an HTTPs-employing website, or your data might just get compromised.

One way we know is by seeing this little lock image on our search bars.

Summary

Thus, HTTP is the method of communication between client (us) and server (website owner). We can technically be both the client and server as well, depending on if we decide to build our own website and put it on the network.
